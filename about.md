## 网站目的

本站服务三个具体场景：

1. 新同学入组后，可以快速找到已有经验贴，少重复踩坑。
2. 在组同学遇到具体问题时，可以按研究方向或事务主题找到可交流的人。
3. 毕业同学可以保留一个轻量连接入口，方便后续和课题组互相找到。

## 内容边界

本站不是正式行政系统，也不是完整通讯录。它只放适合公开展示的信息：

- 经验贴、流程记录、资源索引。
- 成员自愿公开的研究方向、去向和可交流主题。
- GitHub、个人主页、学术主页等低敏联系方式。

不建议放手机号、微信号、身份证明、未公开项目材料、敏感数据或未经本人同意的联系方式。

## 如何参与维护

如果你想新增经验贴，可以复制经验贴模板，写成 Markdown 后提交 pull request；不熟悉 GitHub 的同学也可以通过 issue 表单提交草稿。

如果你想新增或修改成员信息，请优先由本人提交；代为提交时，需要先确认对方同意公开哪些字段。

维护者合并更新后，GitHub Pages 会自动发布新版网站。
